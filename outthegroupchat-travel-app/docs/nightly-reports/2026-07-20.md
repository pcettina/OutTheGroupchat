## Bugs Found
**Settings page always rendered empty notification preferences.** `GET /api/users/notification-preferences` returns `{success, data:{preferences}}`, but `NotificationPreferencesForm` passed that envelope into a parser guarded by `if (!Array.isArray(raw)) return []`. The check always failed, so saved values (DAILY_PROMPT enabled state, schedule, perMemberTargets) never rendered, and saving from that state could write defaults back over them. Now unwrapped defensively.
