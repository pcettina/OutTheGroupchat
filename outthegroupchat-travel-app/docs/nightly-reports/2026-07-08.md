### Drop-in durable fix (Rec #2) — GitHub Action `.github/workflows/auto-ff-main.yml`
```yaml
name: Auto fast-forward main from latest green nightly
on:
  schedule: [{ cron: "0 13 * * *" }]   # 08:00 ET, after nightly completes
  workflow_dispatch:
jobs:
  ff-merge:
    runs-on: ubuntu-latest
    permissions: { contents: write }
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - name: Fast-forward main to newest linear-ancestor nightly
        run: |
          git fetch origin --prune
          latest=$(git branch -r | grep 'origin/nightly/' | sort | tail -1 | tr -d ' ')
          if git merge-base --is-ancestor origin/main "$latest" \
             && [ "$(git rev-list --count origin/main..$latest)" -gt 0 ]; then
            git checkout main
            git merge --ff-only "$latest"
            git push origin main
          else
            echo "No clean linear ff available; skipping."
          fi
```
This ends the recurring stall for good: the agent produces green nightlies, the Action lands them on
`main` every morning, and the escalation guard never trips because `main` is never >1 behind.
