## Bug Found and Fixed

`formatSignalCount()` in `src/app/topics/topicsPageLogic.ts` guarded `count <= 0` **before** flooring, so any fractional count in `(0, 1)` fell through to the plural branch and rendered a bare **"0 Crew signaled"** instead of "Be the first to signal". A Wave-1 test asserted the correct behavior, failed, and was deliberately not relaxed; the implementation now floors first, then guards.

## Security Findings

Clean overall: 0 hardcoded secrets, 0 routes missing Zod (all 66 live routes validate `request.json()` through a Zod schema), no email exposure in `/api/search`, and the only raw query is a constant `SELECT 1` health probe with no interpolation. Two items worth a follow-up:

- **`GET /api/feed/comments` has no auth check** (POST and DELETE are guarded). An unauthenticated caller can enumerate `activityId`/`tripId` and read all comment text plus commenter id/name/image.
- **`/api/crew/[id]` returns member `email`** in the crew detail payload — authenticated and crew-scoped, but broader PII than every other projection in the app.

Unauthenticated auth/beta/cron routes were reviewed and are intentional (pre-session flows, all Zod-validated; cron routes fail closed on a missing `CRON_SECRET`).
